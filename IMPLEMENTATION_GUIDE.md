## How It Works

### Auto-Generate Flow

```
User adds mapping
     ↓
Hook detects change (useAutoPopulateRequestBody)
     ↓
Check: Is body manually edited?
     ├─ No → Generate new template
     └─ Yes → Do nothing (preserve user's edits)
     ↓
Update request body in store
     ↓
CodeEditor shows updated template
```

### Dirty Tracking

```
Initial: body = "{}"  (auto-generated)
     ↓
Add mapping → body = '{"zip":"[postal_code]"}'  (still auto)
     ↓
User types → body = '{"zip":"[postal_code]","custom":"value"}'
     ↓
Hook detects: body ≠ expected → Mark as manually edited
     ↓
Add another mapping → No update (preserve user's work)
     ↓
User clicks "Reset" → Back to auto-generated mode
```

### Option 2: Manual Button Only

Add just a "Generate Template" button:

```tsx
import { generateRequestBody } from '@/lib/request-body-generator'

// In component
const handleGenerate = () => {
  const template = generateRequestBody(mappings, contentType)
  updateBody(template)
}

<Button onClick={handleGenerate}>
  Generate Template from Mappings
</Button>
```

**Pros:**
- ✅ Simple
- ✅ User has full control

**Cons:**
- ❌ User must click button each time
- ❌ No dirty tracking

### Option 3: Auto-Generate Only on Save

Generate template when user saves config:

```tsx
const handleSave = () => {
  // Before saving, regenerate templates if auto-generated
  if (isAutoGeneratedBody(config.ping.requestBody.body, mappings, contentType)) {
    const newBody = generateRequestBody(mappings, contentType)
    updatePingRequestBody(newBody)
  }

  // Save config...
}
```

### Disable Auto-Populate for POST

```typescript
// In request-body-settings.tsx
const { isManuallyEdited, resetToAutoGenerated } = useAutoPopulateRequestBody(phase, {
  enabled: phase === 'ping' // Only auto-populate PING, not POST
})
```

### Custom Template Format

Add new format in `request-body-generator.ts`:

```typescript
export function generateRequestBody(mappings, contentType): string {
  switch (contentType) {
    case 'application/graphql':
      return generateGraphQLTemplate(mappings)

    // ... existing cases
  }
}

function generateGraphQLTemplate(mappings: FieldMapping[]): string {
  const fields = mappings.map(m => `${m.name}`).join('\n    ')
  return `query {
  lead {
    ${fields}
  }
}`
}
```

## Troubleshooting

### Body not updating

**Check 1:** Is it marked as "Manually edited"?
- Click "Reset to Auto-generated"

**Check 2:** Is hook enabled?
```typescript
useAutoPopulateRequestBody(phase, { enabled: true })
```

**Check 3:** Are mappings actually changing?
- Console log in hook to verify

### XML tags look weird

This is correct! Invalid XML characters are sanitized:
- `first-name` → `first_name` (hyphen → underscore)
- `123field` → `_123field` (number prefix → underscore)

### Want to keep user edits when adding mappings

This is by design - dirty tracking prevents data loss.

If you want to merge:
1. User manually edits
2. User adds new mapping
3. Click "Reset" → Loses manual edits ❌

Alternative: Add "Sync Mappings" button that adds new fields to existing template.
