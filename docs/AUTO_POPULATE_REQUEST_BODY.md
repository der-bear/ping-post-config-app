## Auto-Populate Request Body Feature

Automatically generates request body templates based on field mappings and selected content-type.

## Overview

When users add field mappings (e.g., `zip` → `postal_code`), the request body template is automatically generated in the appropriate format (JSON, XML, or URL-encoded). This saves time and ensures consistency.

### Features

1. **Auto-generate on mapping changes** - Template updates when fields are added/removed
2. **Format-aware** - Generates JSON, XML, or URL-encoded based on content-type
3. **Smart dirty tracking** - Stops auto-updating once user manually edits
4. **One-click reset** - Reset to auto-generated template anytime
5. **Sample data preview** - Generate template with mock values for testing

## How It Works

### Example Flow

**Step 1: User adds mappings**
```
Field Mappings:
- zip → postal_code
- ip → ip_address
- first_name → f_name
```

**Step 2: Content-Type: JSON (auto-generated)**
```json
{
  "zip": "[postal_code]",
  "ip": "[ip_address]",
  "first_name": "[f_name]"
}
```

**Step 3: User switches to XML → Auto-regenerates**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<request>
  <zip>[postal_code]</zip>
  <ip>[ip_address]</ip>
  <first_name>[f_name]</first_name>
</request>
```

**Step 4: User manually edits → Auto-update stops**
```xml
<!-- User adds custom field manually -->
<request>
  <zip>[postal_code]</zip>
  <ip>[ip_address]</ip>
  <first_name>[f_name]</first_name>
  <custom_field>hardcoded_value</custom_field>
</request>
```

UI now shows: `"Manually edited"` with `"Reset to Auto-generated"` button

## Implementation

### 1. Request Body Generator (`src/lib/request-body-generator.ts`)

Core logic for generating templates:

```typescript
generateRequestBody(mappings, contentType)
// Returns template string based on format

generateSampleRequestBody(mappings, contentType)
// Returns template with mock data filled in

// Checks if body matches auto-generated template
```

### 2. Auto-Populate Hook (`src/hooks/use-auto-populate-body.ts`)

React hook that manages auto-population:

```typescript

```

**Logic:**
- Monitors mappings and content-type changes
- Generates new template when they change
- Tracks if user has manually edited
- Stops auto-updating once manually edited
- Provides reset function

### 3. Toolbar Component (`src/features/delivery-method/components/request-body-toolbar.tsx`)

UI controls:

```tsx
<RequestBodyToolbar
  isAutoGenerated={!isManuallyEdited}
  onReset={resetToAutoGenerated}
  onUpdate={updateBody}
/>
```

**Shows:**
- ✨ "Auto-generated from field mappings" (when auto)
- "Manually edited" (when user-edited)
- "Reset to Auto-generated" button (when manual)
- "Preview with Sample Data" button (always)

## Supported Formats

### JSON (application/json)
```json
{
  "zip": "[postal_code]",
  "ip": "[ip_address]"
}
```

### XML (application/xml, text/xml, application/soap+xml)
```xml
<?xml version="1.0" encoding="UTF-8"?>
<request>
  <zip>[postal_code]</zip>
  <ip>[ip_address]</ip>
</request>
```

**Note:** XML tag names are sanitized (invalid chars → `_`)

### URL-Encoded (application/x-www-form-urlencoded)
```
zip=[postal_code]&ip=[ip_address]
```

### Other formats
Falls back to JSON template.

## Troubleshooting

**Q: Body not auto-updating when I add mappings**
- Check if it's marked as "Manually edited"
- Click "Reset to Auto-generated"
- Verify `enabled: true` in useAutoPopulateRequestBody

**Q: XML tags have underscores**
- This is correct - invalid XML chars are sanitized
- `first-name` becomes `first_name`
- `123field` becomes `_123field`

**Q: Want nested JSON objects**
- Currently not supported (flat only for demo)
- Future enhancement planned

**Q: Sample data not accurate**
- Edit `getMockValue()` function
- Or set `testValue` in field mappings

## Architecture Decisions

### Why Auto-Populate?

**Problem:** Users had to manually write templates, easy to make mistakes:
```json
{
  "zip": "[postal_code]",  // ✅ Correct
  "ip": "[ip_addres]"      // ❌ Typo! Should be ip_address
}
```

**Solution:** Generate from mappings = always correct:
```typescript
mappings.forEach(m => obj[m.name] = `[${m.mappedTo}]`)
```

### Why Track Dirty State?

**Without tracking:**
- User adds custom field manually
- Add new mapping → Template regenerates
- Custom field lost! 😞

**With tracking:**
- User edits manually → Auto-update stops
- Add mapping → Template stays as-is
- User decides when to reset ✅

### Why Support Multiple Formats?

Different buyers expect different formats:
- **JSON** - Most common, easy to parse
- **XML** - Enterprise systems, SOAP APIs
- **URL-encoded** - Form submissions, legacy systems

Auto-generate all formats = works for any buyer.
